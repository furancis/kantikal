# Goal: Suno Visual Studio

Build and preserve the full product shape for a dynamic visual Suno music generation app.

## Target Result

A repo exists with PRD, visual React MVP, API coverage ledger, Greptile rules, verification suite, and GoalBuddy state.

## Guardrails

- Visual app first.
- CLI is a hidden engine only.
- Music video is a subfeature of the overall Suno workflow.
- Perfect lipsync is a hard export gate.
- Full API coverage never disappears.
- Destructive cleanup is archive-first with receipts.
- Secrets stay server-side.

## Acceptance

- First viewport communicates visual workflow app.
- Canvas contains brief, lyrics, style, voice, generation batch, chosen track, stems, MV lane, and release pack.
- Tests lock the MV boundary and full-shape anchors.
- Greptile rules mirror these guardrails.
- Full suite passes.
